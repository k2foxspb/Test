Windows
    Установка:
        -скачиваем проект, устанавливаем виртуальное окружение:
           "python -m venv venv":
           " venv\Scripts\activate.bat"
        -в виртуальном окружении:
            "pip install -r requirements.txt"
        -или в глобальном окружении:
            "pip install pyserial"
    Запуск:
        -открываем последовательный виртуальный порт, в файле var.py устанавливаем COM_PORT_1, COM_PORT_2
        -в папке проекта:
            "python launcher.py"
            либо запустить каждое приложение отдельно:
            "python m1.py" "python m2.py" "python m3.py"


    Возможные ошибки:
        в файле launcher изменить ('py m1.py', 'py m2.py', 'py m3.py') на ('python m1.py', 'python m2.py' 'python m3.py')

Принцип работы:

-В приложение М3 вводим цифру от 0 до 99(если больше то длина пакета увеличится).
 -М3 отправляет в М2 пакет данных.
 -М2 получает пакет данных от М3 и пересылает в М1.
 -М1 при подключении М2 выставляет таймер "SERVER_INTERVAL" и меняет флаг "EXCHANGE". Если "SERVER_INTERVAL" больше
     заданного и флаг=1, М1 отвечает пакетом данных в М2 изменив B2_WRITE на B2_ANSWER
 -М2 получает пакет от М1 и пересылает пакет в М3
 -М3 получает пакет данных от М2 и цикл повторяется

 -Если ввести в командную строку приложения М3 "65535" : М3 отправит пакет 0xffff в М2, М2 в М1, М1 отправит сигнал
  на закрытие портов и сам завершит работу. М2 получает сигнал на закрытие, отправляет его в М3 и завершает работу.
  М3 получает сигнал и завершает работу

 -Если ввести в командную строку приложения М3 "65530" - сервер М1 изменит флаг EXCHANGE и перестанет отправлять пакеты в М2


 М1 общается через сокет 'localhost' порт 7777
 М2 открывает сокет и последовательный порт
 М3 слушает и записывает только последовательный порт
